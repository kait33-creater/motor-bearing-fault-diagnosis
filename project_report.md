## 1. 项目摘要

本项目面向工业电机运行状态监测场景，围绕电机运行过程中常见的电压、电流、温度、振动、转速和负载率等关键参数，构建了一套基于 Python 的电机运行状态监测与故障预警系统。

项目首先通过规则生成方式构造电机运行模拟数据，并完成数据清洗、探索性数据分析和滑动窗口特征工程；随后分别建立了规则阈值预警模型和机器学习故障识别模型，实现对电机“正常、预警、故障”三类运行状态的识别，以及对“过载、过热、轴承异常、电压波动”等典型故障类型的判断。最后，基于 Streamlit 搭建可视化看板，实现电机状态展示、参数趋势分析、故障类型预测、风险得分显示和维护建议输出。

实验结果表明，规则阈值模型具有较强的可解释性，适合作为工程基线方法；Random Forest 模型在模拟数据集上取得了较高识别效果，其中运行状态识别准确率达到 **0.9683**，故障类型识别准确率达到 **0.9733**。需要说明的是，本项目当前使用模拟数据，模型结果主要用于验证方法流程的完整性和可行性，后续仍需接入真实工业数据进一步验证模型泛化能力。

## 2. 项目背景

电机是工业生产系统中的核心动力设备，广泛应用于生产线、泵站、风机、压缩机、输送设备和自动化装备中。电机在长期运行过程中，可能受到负载变化、电压波动、散热条件、机械磨损和轴承状态等因素影响，出现过载、过热、振动异常、转速波动等问题。如果不能及时发现异常状态，可能会导致设备停机、生产中断，甚至引发安全事故。

传统电机维护方式通常依赖人工巡检和定期检修，存在实时性不足、判断依赖经验、故障预警滞后等问题。随着工业数字化、智能制造和工业互联网的发展，基于数据采集、状态监测和智能算法的设备故障预警逐渐成为电气自动化领域的重要方向。

本项目尝试将电气自动化专业场景与 Python 数据分析、规则预警和机器学习方法结合，构建一个小型但完整的电机运行状态监测系统，为后续开展工业数据分析、智能制造、自动化测试和设备状态监测方向的学习与实践打基础。

## 3. 项目目标

本项目的总体目标是构建一个完整的电机运行状态监测与故障预警系统，实现从数据生成到可视化展示的完整闭环。

具体目标包括：

1. 构建电机运行模拟数据集，包含电压、电流、温度、振动、转速、负载率等关键运行参数；

3. 完成探索性数据分析，绘制运行状态分布、故障类型分布和关键参数趋势图；
4. 构造滑动窗口统计特征，为规则模型和机器学习模型提供输入；
5. 建立规则阈值预警模型，实现状态判断、故障类型识别和维护建议输出；
6. 建立机器学习分类模型，对比 Logistic Regression 和 Random Forest 的识别效果；
7. 基于 Streamlit 搭建可视化看板，实现项目演示和结果展示；
8. 整理项目报告、运行说明和后续改进方向。

## 6. 数据设计

### 6.1 数据字段说明

本项目构造的数据集共包含 10 个基础字段：

| 字段名 | 含义 | 说明 |
|---|---|---|
| timestamp | 采样时间 | 每 5 分钟采样一次 |
| motor_id | 电机编号 | 包含 M001、M002、M003 三台电机 |
| voltage | 电压 | 单位 V |
| current | 电流 | 单位 A |
| temperature | 温度 | 单位 ℃ |
| vibration | 振动强度 | 模拟振动指标 |
| speed | 转速 | 单位 rpm |
| load_rate | 负载率 | 单位 % |
| status | 运行状态 | 正常、预警、故障 |
| fault_type | 故障类型 | 正常、过载、过热、轴承异常、电压波动 |

### 6.2 故障类型设计逻辑

数据生成时，根据典型电机异常场景设置不同参数变化规律：

| 故障类型 | 参数变化特征 | 工程含义 |
|---|---|---|
| 正常 | 电压、电流、温度、振动、转速、负载率均在正常范围内波动 | 设备处于稳定运行状态 |
| 过载 | 电流升高、负载率升高，温度可能上升 | 电机承受负载过大 |
| 过热 | 温度明显升高，可能伴随电流升高 | 散热不足或绕组温升异常 |
| 轴承异常 | 振动增强，转速波动增大 | 轴承磨损、润滑不良或机械安装异常 |
| 电压波动 | 电压偏离正常范围，电流可能不稳定 | 供电侧存在不稳定因素 |

### 6.3 数据规模与分布

本项目共生成 3 台电机的运行数据，每台电机 1000 条记录，总计：

```text
数据规模：3000 行 × 10 列
```

最终状态分布为：

| 运行状态 | 样本数量 |
|---|---:|
| 正常 | 2005 |
| 预警 | 708 |
| 故障 | 287 |

最终故障类型分布为：

| 故障类型 | 样本数量 |
|---|---:|
| 正常 | 2005 |
| 过载 | 334 |
| 过热 | 291 |
| 轴承异常 | 270 |
| 电压波动 | 100 |

该分布符合设备运行监测场景中“正常样本较多、异常样本较少”的基本特征，同时保留了一定数量的预警和故障样本，能够支撑后续建模分析。

## 7. 数据清洗

### 7.1 清洗目标

数据清洗阶段的主要目标是保证后续分析与建模输入数据的完整性和一致性，具体包括：

1. 读取原始数据文件 `data/raw/motor_data.csv`；
2. 检查数据规模、字段类型和缺失值；
3. 将 `timestamp` 字段转换为时间格式；
4. 按照 `motor_id` 和 `timestamp` 进行排序；
5. 对电压、电流、温度、振动等数值字段的缺失值进行填充；
6. 删除重复行；
7. 保存清洗后的数据文件。

### 7.2 缺失值处理方法

本项目在模拟数据中人为加入了少量缺失值，用于模拟传感器短时采集异常或数据传输缺失情况。清洗时采用以下策略：

1. 先按电机编号分组进行前向填充；
2. 如果某台电机开头仍有缺失值，则使用同一电机该字段均值进行填充；
3. 若仍存在极少数缺失，则使用全局均值兜底填充。

清洗后缺失值统计如下：

```text
timestamp      0
motor_id       0
voltage        0
current        0
temperature    0
vibration      0
speed          0
load_rate      0
status         0
fault_type     0
```

清洗后数据保存为：

```text
data/processed/motor_data_cleaned.csv
```

## 9. 特征工程设计

### 9.1 特征工程目的

原始数据仅包含瞬时采样值，而电机故障往往不仅表现为某一时刻的异常，还可能表现为一段时间内的持续升高、波动增强或趋势变化。因此，本项目基于滑动窗口构造统计特征，用于增强模型对异常状态的识别能力。

### 9.2 滑动窗口设置

本项目采样间隔为 5 分钟，设置滑动窗口大小为 6，即：

```text
6 个采样点 × 5 分钟 = 30 分钟
```

因此，部分特征反映的是近 30 分钟内的运行状态。

### 9.3 构造特征列表

本项目在原始字段基础上新增了 14 个特征，最终特征数据共 24 列。

| 特征名 | 含义 |
|---|---|
| current_mean_30min | 近 30 分钟电流均值 |
| current_std_30min | 近 30 分钟电流标准差 |
| temperature_mean_30min | 近 30 分钟温度均值 |
| temperature_max_30min | 近 30 分钟温度最大值 |
| temperature_change | 相邻采样点温度变化量 |
| vibration_mean_30min | 近 30 分钟振动均值 |
| vibration_max_30min | 近 30 分钟振动最大值 |
| speed_mean_30min | 近 30 分钟转速均值 |
| speed_std_30min | 近 30 分钟转速标准差 |
| voltage_mean_30min | 近 30 分钟电压均值 |
| voltage_std_30min | 近 30 分钟电压标准差 |
| load_rate_mean_30min | 近 30 分钟负载率均值 |
| current_load_ratio | 电流与负载率比值 |
| temperature_load_ratio | 温度与负载率比值 |

特征工程后数据保存为：

```text

```

## 10. 规则阈值预警模型

### 10.1 模型设计思想

规则阈值模型基于电气工程经验和设备运行参数阈值进行判断。该方法不依赖训练数据，具有较强可解释性，适合作为设备状态监测系统中的工程基线模型。

模型从以下 6 类因素计算风险得分：

1. 电流异常；
2. 温度异常；
3. 振动异常；
4. 电压异常；
5. 负载率异常；
6. 转速异常。

每出现一类异常，风险得分加 1。

### 10.2 状态判定规则

本项目采用如下状态判定逻辑：

| 风险得分 | 规则判断状态 |
|---:|---|
| 0 | 正常 |
| 1 | 预警 |
| ≥ 2 | 故障 |

该设计考虑到在设备故障预警场景中，如果同时出现两个及以上关键指标异常，应尽量避免漏报故障风险。

### 10.3 故障类型判断规则

故障类型根据主要异常参数进行判断：

| 故障类型 | 判断依据 |
|---|---|
| 过载 | 电流偏高且负载率偏高 |
| 过热 | 温度明显升高或持续高温 |
| 轴承异常 | 振动升高或转速波动明显 |
| 电压波动 | 电压明显越界或波动较大 |
| 正常 | 未触发异常规则 |

### 10.4 规则模型结果

规则模型输出结果如下：

| 指标 | 结果 |
|---|---:|
| 状态识别准确率 | 0.7350 |
| 故障类型识别准确率 | 0.7650 |

规则模型预测状态分布为：

| 规则状态 | 样本数量 |
|---|---:|
| 正常 | 1775 |
| 预警 | 962 |
| 故障 | 263 |

规则模型预测故障类型分布为：

| 规则故障类型 | 样本数量 |
|---|---:|
| 正常 | 1775 |
| 轴承异常 | 524 |
| 过热 | 370 |
| 过载 | 250 |
| 电压波动 | 81 |

### 10.5 规则模型分析

规则阈值模型的优势在于逻辑清晰、可解释性强，能够直接说明某一异常状态是由哪些参数触发的。例如，温度升高可对应过热风险，振动增强可对应轴承异常风险，电压越界可对应电压波动风险。

但规则模型也存在局限：阈值设定依赖人工经验，不同设备、工况和环境下阈值可能需要重新调整；同时，当多个参数存在复杂耦合关系时，规则模型难以充分捕捉多维特征之间的非线性关系。因此，本项目进一步引入机器学习模型进行对比。

2. **故障类型识别：** 预测故障类型为正常、过载、过热、轴承异常或电压波动。

### 11.3 模型选择

| 模型 | 作用 |
|---|---|
| Logistic Regression | 基础对照模型 |
| Random Forest | 主要机器学习模型 |

Logistic Regression 结构简单、可作为基础基线；Random Forest 能够处理非线性关系，并对多维特征具有较强适应能力，因此作为本项目的主要模型。

### 11.4 训练与测试集划分

采用训练集和测试集划分方式进行模型评估：

```text
训练集：80%
测试集：20%
```

为保证不同类别在训练集和测试集中的比例基本一致，使用分层抽样方式进行划分。

### 11.5 模型结果

运行状态识别结果如下：

| 模型 | 准确率 |
|---|---:|
| Logistic Regression | 0.8250 |
| Random Forest | 0.9683 |

故障类型识别结果如下：

| 模型 | 准确率 |
|---|---:|
| Logistic Regression | 0.8600 |
| Random Forest | 0.9733 |

可以看出，Random Forest 在运行状态识别和故障类型识别两个任务上均明显优于 Logistic Regression。

### 11.6 混淆矩阵分析

![运行状态识别混淆矩阵](outputs/figures/ml_status_confusion_matrix.png)

运行状态识别混淆矩阵显示，Random Forest 对正常状态识别效果较好，对故障状态也具有较高识别能力，少量误判主要发生在正常与预警、预警与故障之间。这符合设备状态变化的连续性特征，即预警状态本身位于正常和故障之间，边界更容易产生混淆。

![故障类型识别混淆矩阵](outputs/figures/ml_fault_confusion_matrix.png)

故障类型识别混淆矩阵显示，Random Forest 对过载、过热、轴承异常和电压波动均具有较好的识别效果。其中电压波动样本数量相对较少，因此其识别结果更容易受到样本规模影响。

### 11.7 机器学习模型分析

机器学习模型相比规则阈值模型具有更高准确率，说明模型能够从多维运行参数和滑动窗口特征中学习状态与故障之间的关系。尤其是 Random Forest 模型，能够处理非线性关系，并在模拟数据集上取得较好分类效果。

不过需要注意的是，本项目当前数据由规则模拟生成，标签与特征之间存在较强关联，因此模型结果不能直接等同于真实工业现场表现。后续如果接入真实传感器数据，还需要重新进行数据标注、模型训练、交叉验证和现场测试。

## 16. 项目不足

本项目仍存在一些不足：

1. **数据为模拟数据**  
   当前数据由规则生成，虽然符合基本工程逻辑，但仍不能完全代表真实工业现场数据。

2. **故障模式相对简化**  
   当前仅包含过载、过热、轴承异常和电压波动四类故障，未覆盖更多复杂工况。

3. **模型泛化能力尚未验证**  
   Random Forest 在模拟数据上效果较好，但尚未在真实设备数据上验证泛化能力。

4. **规则阈值依赖人工设定**  
   阈值设置具有经验性，不同型号电机和不同运行环境下可能需要重新调整。

5. **看板交互功能仍可增强**  
   当前看板主要用于展示已有数据，后续可增加 CSV 上传、实时刷新、报警记录导出等功能。
